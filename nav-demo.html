Navigation demo

Accessible header + nav (desktop + mobile)

This file demonstrates the shared header pattern you’ll reuse across all pages. On desktop, links are visible in the header. On mobile, an accessible menu opens/closes with a button, supports keyboard use, and closes with Escape or by clicking the dim overlay.

What to test

1) Desktop: widen window ≥ 860px — links appear, hamburger disappears.
2) Mobile: narrow window — hamburger appears, links hide, menu opens/closes.
3) Keyboard: press Tab to reach menu button, press Enter to open, then tab through links.
4) Close: press Esc or click outside on the dim overlay.

Notes

The logo/wordmark is a placeholder (MK + “Michael Kai”). You can swap it later without changing layout. All colors, borders, and type styles match your design-system tokens.
